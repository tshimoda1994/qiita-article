## はじめに

この記事は、実務での開発経験がない私が Go 言語を習得するために、毎週ひとつツールを公開していく試みの第 1 回です。
今回は TCP ポートスキャナー `pscan` を作成しました。

## 読んで得られること

- ほぼGo の標準ライブラリだけで CLI ツールを作る流れ
- goroutine と Worker Pool を使った並列処理のイメージ
- 同じように Go 言語を練習している方の参考になればと思います

## 試したいと思った理由

既存のツールであれば仕様や実装の参考情報が多く、Go 言語の初学者でも取り組みやすいと考えました。
また、週次でツールを公開することで、継続的に学習を続けられる仕組みにしたいという思いから、この取り組みを始めました。

## プロダクト概要

| 項目 | 内容 |
|------|------|
| 名前 | pscan |
| 一言で | Go 製の TCP ポートスキャナー |
| リンク | [リポジトリ](https://github.com/tshimoda1994/pscan) |

## できること

- 指定ホストに対して TCP Connect Scan を実行できます
- Well-Known ポート（1〜1024）の範囲で、開いているポートを表示します
- `-start` / `-end` でスキャンするポート範囲を指定できます
- `-timeout` で接続タイムアウトを指定できます
- Worker Pool により、並列数を抑えつつ高速にスキャンします

## インストール / 試し方

```bash
# リポジトリの取得
git clone https://github.com/tshimoda1994/pscan.git
cd pscan

# ビルド
go build -o pscan pscan.go

# 実行（ホスト名を引数で指定）
./pscan localhost
```

実行例:

```text
port 22 open
port 80 open
port 443 open
```

ポート範囲やタイムアウトを指定する例:

```bash
./pscan -start 1 -end 100 -timeout 500ms localhost
./pscan -host google.com
```

## モチベーション（なぜ作ったか）

ポートスキャナーはネットワークの基礎を学ぶうえで題材にしやすく、`net` パッケージや goroutine、チャネルなど Go の基本要素をまとめて触れられると考えました。
あわせて、外部ライブラリに頼らず標準ライブラリ中心で実装することで、理解の範囲を自分の手の届くところに抑えたいという意図もあります。

## 技術スタック

- Go 1.26.3
- `net`（TCP 接続）
- goroutine / Worker Pool
- Go 標準ライブラリのみ（外部依存なし）

## まとめ

- 標準ライブラリを中心に使うことで、インプット量を抑えつつ CLI ツールを作成できました
- Worker Pool を使うことで、ポートごとに goroutine を乱立させずに並列スキャンできることを確認しました
- 改良の余地はありますが、あくまで自分の理解の範囲で作ることを意識したことで、コード全体を追える状態にできました
- 有識者の方おりましたら、ぜひアドバイスやフィードバックをいただけると励みになります
